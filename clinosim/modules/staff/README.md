# staff

Healthcare staff generation and assignment. Creates realistic staff rosters with Japanese names and assigns staff to clinical events.

## Public API

```python
from clinosim.modules.staff.engine import (
    generate_roster,   # (hospital_scale, country, rng) -> StaffRoster
    assign_staff,      # (event_type, department, roster, rng) -> dict[str, str]
    StaffMember,
    StaffRoster,
)
```

### `generate_roster(hospital_scale, country, rng) -> StaffRoster`
Generates a complete staff roster for a medium JP hospital: 10 physicians, 30 nurses, 10 lab techs, 4 radiologists, 8 pharmacists. Japanese names from weighted surname/given name lists.

### `assign_staff(event_type, department, roster, rng) -> dict[str, str]`
Assigns appropriate staff to clinical events. Returns `{role: staff_id}` mapping.

## How to add staff for a new department

Staff are generated inside the `generate_roster()` function in `engine.py`. The function
currently dispatches on `(hospital_scale, country)`. To add a new department:

1. Inside the relevant `if hospital_scale == "..." and country == "..."` block, append a
   loop that creates `StaffMember` objects with the new department name:
   ```python
   # Surgery: 6 physicians, 20 nurses
   for i in range(6):
       name = _generate_jp_name("M" if i % 4 != 0 else "F", rng)
       roster.members.append(StaffMember(
           staff_id=f"DR-SG-{i+1:03d}",
           name=name,
           role="physician",
           department="surgery",
           specialty=["general_surgery", "orthopedics", ...][i],
           qualification_year=int(rng.integers(1990, 2020)),
       ))
   ```
2. Add the corresponding `case` branch in `assign_staff()` for any event types that should
   route to the new department's staff:
   ```python
   case "pre_op_assessment" | "post_op_rounds":
       physicians = roster.get_by_role("physician", "surgery")
       ...
   ```
3. The `StaffRoster.get_by_role(role, department)` helper filters by both fields, so
   no further changes are needed for basic assignment logic.

## How name generation works via locale

For Japanese staff, names are generated by `_generate_jp_name(sex, rng)` in `engine.py`,
which samples from two hardcoded lists (`JP_SURNAMES`, `JP_GIVEN_M`, `JP_GIVEN_F`) using
uniform probability.

For population persons (which use the same locale system), the full locale-aware pipeline
is used instead:

1. `_load_name_data(country)` calls `clinosim.locale.loader.load_names(country)`, which
   reads `clinosim/locale/{country}/names.yaml`.
2. `_sample_surname()` draws from the `surnames` list weighted by the `weight` field.
3. `_sample_given_name()` draws from `given_names_male` or `given_names_female`, also
   weighted.
4. Household surname rules (shared / mostly_shared / not_shared) are read from
   `clinosim/locale/shared/naming_rules.yaml` and applied per-member inside
   `generate_population()`.

To support locale-aware staff names for a new country, replace the hardcoded
`_generate_jp_name()` call with the same `_load_name_data` / `_sample_surname` /
`_sample_given_name` pipeline, and add the country's name YAML under
`clinosim/locale/{country}/names.yaml`.
