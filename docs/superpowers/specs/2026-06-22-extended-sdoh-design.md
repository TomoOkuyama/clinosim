# Extended SDOH (smoking, alcohol, JP 要介護度) — AD-55 Base

**Date:** 2026-06-22
**Status:** approved (design)
**Type:** AD-55 Base data enrichment (always-on)

## Goal

Emit three social-history / SDOH `Observation`s, extending the existing occupation
Observation (US Core, LOINC 11341-5): tobacco smoking status, alcohol use, and the
Japanese long-term-care need level (要介護度). Completes the AD-55 Base roadmap.

## Components

### 1. Tobacco smoking status (both countries)

`PatientProfile.smoking_status` (`never`/`former`/`current`) is already generated.
Emit a US Core Smoking Status Observation:
- category: social-history; code: LOINC **72166-2** "Tobacco smoking status".
- valueCodeableConcept: SNOMED — never → `266919005` "Never smoked tobacco",
  former → `8517006` "Ex-smoker", current → `449868002` "Current every day smoker"
  (US Core Smoking Status value set; high confidence).
- Read from `ctx.patient_data.get("smoking_status")`; patient-level, de-duped.

### 2. Alcohol use (both countries)

`PatientProfile.alcohol_use` (`none`/`social`/`heavy`) is already generated. Emit a
social-history Observation:
- code: LOINC **11331-6** "History of alcohol use".
- valueCodeableConcept: SNOMED per level (candidates `105542008` non-drinker,
  `160573003` social drinker, `86933000` heavy drinker — **`# TODO: verify`**, the
  SNOMED API is unreachable from this environment).
- Read from `ctx.patient_data.get("alcohol_use")`; patient-level, de-duped.

### 3. JP long-term-care need level / 要介護度 (JP only, new attribute)

The 介護保険 certification: 自立 / 要支援1-2 / 要介護1-5. **Age-driven** (rare < 65,
rising sharply ≥ 75), JP only.
- **Generated by an enricher** (`modules/care_level/`, JP-only, `post_records`,
  **`person_id` sub-seed** so the master stream is unperturbed, AD-16). Age → level
  distribution from `locale/jp/care_level_rates.yaml`. Stored on
  `CIFPatientRecord.care_level: str` (a `jp-care-level` code; `""` when not certified
  / non-JP).
- Emit a social-history Observation: valueCodeableConcept from a new **`jp-care-level`
  code system** (MHLW 介護保険 区分 — authoritative classification; no clean
  international code, so a documented local code system is minted). Observation.code
  = a care-need-level concept (LOINC/SNOMED candidate, **`# TODO: verify`**).

## Architecture (mirrors existing Base patterns)

- **`clinosim/modules/output/_fhir_sdoh.py`**: `_build_smoking_status(ctx)`,
  `_build_alcohol_use(ctx)`, `_build_care_level(ctx)` — all social-history
  Observations, patient-scoped ids (`smoking-{pid}`, `alcohol-{pid}`,
  `carelevel-{pid}`) so the write-time de-dup keeps one per patient. Registered in
  `_BUNDLE_BUILDERS`.
- **`clinosim/modules/care_level/`**: `engine.py` (pure, seeded
  `assign_care_level(age, country, rng) -> str` returning the `jp-care-level` code or
  `""`), `reference_data/care_level.yaml` (levels → code/display, age bands);
  `enricher.py` (`post_records`, JP-only via `enabled=lambda c: c.country == "JP"`,
  `derive_sub_seed(master, _CL_SEED_OFFSET, person_id)`).
- **`clinosim/locale/jp/care_level_rates.yaml`**: age-band → level weights.
- **`clinosim/types/output.py`**: add `care_level: str = ""` to `CIFPatientRecord`.
- **Codes**: add LOINC `72166-2`, `11331-6` to `loinc.yaml`; SNOMED smoking +
  alcohol values to `snomed-ct.yaml`; new `clinosim/codes/data/jp-care-level.yaml`
  (8 levels, EN + JA, source MHLW) + a `jp-care-level` system URI registered in
  `codes/loader.py` `_BUILTIN_URIS`.
- **CSV**: add `alcohol_use` to the patients row (smoking already present); new
  `care_level.csv` (patient_id, code, display) for JP.

## Determinism (AD-16)

Smoking/alcohol read already-generated attributes — **no new RNG draw**. Care level
uses a `person_id` sub-seed — no master draw. Byte-diff (same seed, master vs
branch): only the new social-history Observations appear in `Observation.ndjson`;
all existing resources byte-identical (US adds smoking+alcohol; JP adds
smoking+alcohol+care-level).
